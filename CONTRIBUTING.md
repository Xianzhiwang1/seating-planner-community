# Contributing

Thanks for helping improve **Seating Planner**! This repository is a public
tracker and showcase. The main application source is private, so contributions
here are mostly **issues, bug reports, feature ideas, and documentation
feedback** rather than application code.

## How to contribute

### Report a bug

1. Search [existing issues](../../issues) to avoid duplicates.
2. Open a new issue using the **Bug report** template.
3. Include clear steps to reproduce, what you expected, and what actually
   happened. Screenshots, console errors, and your browser/OS help a lot.

### Request a feature

1. Search existing issues and discussions first.
2. Open a new issue using the **Feature request** template.
3. Describe the problem you're trying to solve, not just the solution — context
   helps us find the best fix.

### Suggest a documentation fix

The showcase docs and snippets in this repo (under `docs/`) are open to
correction. Open an issue, or a pull request if you'd like to propose the exact
wording.

## Ground rules

- Be respectful and constructive. Assume good intent.
- One issue per report — it keeps discussion focused.
- Do **not** post secrets, API keys, real student data, or other personal
  information in issues. Redact before sharing logs or screenshots.

Please do **not** open public issues for security vulnerabilities. If a
`SECURITY.md` is present, follow its instructions; otherwise contact the
maintainer privately so the issue can be addressed before disclosure.
